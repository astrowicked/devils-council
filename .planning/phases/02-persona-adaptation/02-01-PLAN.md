<objective>
Adapt the 4 core persona agents + Chair for OpenCode's execution model, and create a `council-review` orchestrator agent that enables single-invocation full council review (labeled experimental).

**Orchestration Strategy Decision: Option E — "Review agent with embedded persona instructions" (EXPERIMENTAL)**

Rationale: OpenCode has no nested subagent spawning. A single `council-review` agent reads the artifact, role-plays each persona sequentially with explicit section boundaries (reducing cross-contamination via structural separation), produces 4 scorecards in the canonical YAML-frontmatter format, then synthesizes as the Chair. Individual persona agents remain available for standalone invocation (user picks `@staff-engineer` for a single-lens review).

**Honest limitation:** This is a best-effort approximation, NOT equivalent to Claude Code's parallel isolated subagents. A single context window means prior persona output is visible to subsequent personas despite "CONTEXT RESET" instructions. Cross-contamination is a known quality risk that can only be truly eliminated by invoking persona agents individually. The council-review agent is a convenience tool for quick reviews; standalone agents are the primary recommended workflow for important artifacts.

**Schema compatibility commitment:** Output uses identical YAML keys (`persona`, `findings` array with `target`/`claim`/`evidence`/`ask`/`severity`/`category`), identical severity enum (BLOCKER/MAJOR/MINOR), and identical section structure as Claude Code. Different LLM outputs will naturally vary in prose content — schema-compatible means structurally parseable by the same tools, not byte-identical.

Purpose: Enable devils-council critique in OpenCode with the same voice differentiation and scorecard schema as Claude Code, despite the single-agent constraint.
Output: 6 adapted agent files + 1 new orchestrator agent + updated build script
</objective>
