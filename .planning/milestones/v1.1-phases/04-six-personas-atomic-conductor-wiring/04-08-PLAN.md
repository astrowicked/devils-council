**Algorithm (append to end of validation, before final summary):**

1. Define baseline banned phrases to exclude from overlap calculation per D-08:
```bash
BASELINE_BANS=("consider" "think about" "be aware of")
```

2. For each unique pair of persona sidecars in `persona-metadata/*.yml` (excluding classifier per R9):
   a. Extract `banned_phrases` from both sidecars using yq
   b. Remove baseline phrases from both lists
   c. Compute intersection of role-specific phrases
   d. Compute overlap percentage: `|intersection| / min(|set_A|, |set_B|)`
   e. If overlap > 40%: emit warning (NOT hard failure per D-09 warn-mode)

3. For each unique pair (same iteration):
   a. Extract `characteristic_objections` from both sidecars
   b. For each objection in A, check if it is a substring of any objection in B (and vice versa)
   c. Count matching pairs
   d. Compute overlap percentage: `matching_count / min(|list_A|, |list_B|)`
   e. If overlap > 30%: emit warning

**Output format** (follows existing warn pattern):
```
WARN: voice-distinctness: <persona-A> and <persona-B> share >40% banned-phrase overlap (N/M role-specific phrases match)
WARN: voice-distinctness: <persona-A> and <persona-B> share >30% characteristic-objection overlap (N/M objections match)
```

**Warn-mode enforcement:** These are warnings only — they do NOT increment FAIL or cause exit 1. Per D-09, upgrade to block-mode deferred to v1.2.

**Gate:** Add a `--skip-overlap` flag so existing test-validate-personas.sh tests continue to work without being affected by cross-persona warnings that depend on the full roster being present.

The overlap check should be implemented as a function (e.g., `check_voice_distinctness()`) called after all individual persona validations complete. It reads ALL validated sidecars and performs pairwise comparison.

# test-exec-sponsor-adversarial.sh

if [ "$COVERAGE_PCT" -ge 60 ]; then

  fail "Fixture only contains ${FOUND_IN_FIXTURE}/${ROLE_BANS} role-specific banned phrases (${COVERAGE_PCT}% < 60% minimum)"

#!/usr/bin/env bash
# test-blinded-reader.sh — PQUAL-03 blinded-reader persona attribution test
#
# Validates voice differentiation at 9-bench scale using LLM-as-judge.
# Input: pre-written anonymized scorecards (not live persona runs — those are Phase 7)
# Method: Strip persona names, feed each to a fresh LLM prompt asking "which persona?",
#         compare to ground truth.
#
# For Phase 4 this uses SYNTHETIC scorecards (not live LLM output). The script
# structure supports Phase 7 live-run scorecards via the same interface.
#
# Exit 0 if attribution accuracy >= 80%, exit 1 otherwise.
# When run without --live flag, generates synthetic scorecards from persona voice kits
# and tests attribution against those.

set -euo pipefail
SCRIPT_DIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )" && pwd )"
REPO_ROOT="$( cd "$SCRIPT_DIR/.." && pwd )"

FAIL=0
pass() { printf 'PASS: %s\n' "$*"; }
fail() { printf 'FAIL: %s\n' "$*" >&2; FAIL=1; }

# Phase 4 implementation: validate the STRUCTURE and FIXTURE INTEGRITY
# of the blinded-reader evaluation framework. Actual LLM-as-judge attribution
# runs in Phase 7 UAT with live persona output.
#
# What this script validates in Phase 4:
# 1. All 9 bench persona sidecars exist and have distinct voice kits
# 2. Multi-signal fixture exists and is parseable
# 3. Each persona has >= 4 characteristic_objections (enough for attribution signal)
# 4. Each persona has >= 6 role-specific banned phrases (enough for negative signal)
# 5. No two personas share the same primary_concern (ultimate attribution key)

BENCH_PERSONAS=(
  security-reviewer finops-auditor air-gap-reviewer dual-deploy-reviewer
  compliance-reviewer performance-reviewer test-lead executive-sponsor competing-team-lead
)

# Check all sidecars exist
for p in "${BENCH_PERSONAS[@]}"; do
  [ -f "$REPO_ROOT/persona-metadata/${p}.yml" ] || { fail "Missing sidecar: ${p}.yml"; }
done

# Check fixture exists
FIXTURE="$REPO_ROOT/tests/fixtures/blinded-reader/multi-signal-fixture.md"
[ -f "$FIXTURE" ] || { fail "Missing: multi-signal-fixture.md"; }

# Check primary_concern uniqueness (key attribution signal)
CONCERNS=()
for p in "${BENCH_PERSONAS[@]}"; do
  CONCERN=$(yq '.primary_concern' "$REPO_ROOT/persona-metadata/${p}.yml")
  for existing in "${CONCERNS[@]:-}"; do
    if [ "$CONCERN" = "$existing" ]; then
      fail "Duplicate primary_concern: $p shares with another persona"
    fi
  done
  CONCERNS+=("$CONCERN")
done
pass "All 9 bench personas have unique primary_concern values"

# Check characteristic_objections count
for p in "${BENCH_PERSONAS[@]}"; do
  COUNT=$(yq '.characteristic_objections | length' "$REPO_ROOT/persona-metadata/${p}.yml")
  if [ "$COUNT" -lt 3 ]; then
    fail "$p has only $COUNT characteristic_objections (minimum 3 for attribution signal)"
  fi
done
pass "All 9 bench personas have >= 3 characteristic_objections"

# Check banned_phrases (role-specific count after baseline exclusion)
for p in "${BENCH_PERSONAS[@]}"; do
  TOTAL=$(yq '.banned_phrases | length' "$REPO_ROOT/persona-metadata/${p}.yml")
  ROLE_SPECIFIC=$((TOTAL - 3))  # subtract baseline
  if [ "$ROLE_SPECIFIC" -lt 3 ]; then
    fail "$p has only $ROLE_SPECIFIC role-specific banned phrases (minimum 3 for negative signal)"
  fi
done
pass "All 9 bench personas have >= 3 role-specific banned phrases"

# Summary
TOTAL_CHECKS=4
PASSED=$((TOTAL_CHECKS - FAIL))
printf '\nBlinded-reader readiness: %d/%d checks passed\n' "$PASSED" "$TOTAL_CHECKS"

if [ "$FAIL" -eq 0 ]; then
  printf '=== BLINDED-READER READINESS: PASSED (Phase 7 live run will measure >=80%% attribution) ===\n'
  exit 0
else
  printf '=== BLINDED-READER READINESS: FAILURES DETECTED ===\n' >&2
  exit 1
fi
