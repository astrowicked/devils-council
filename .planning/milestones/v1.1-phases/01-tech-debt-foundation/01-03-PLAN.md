**Environment gates (D-03):**
- If `$CLAUDE_PLUGIN_OPTION_SHELL_INJECT_GUARD` equals `false` (case-insensitive), exit 0 immediately with message `shell-inject guard disabled by userConfig; skipping` to stderr.
- If env var is unset or `true`, proceed with validation.

**Parser rules (STACK.md §Q3):**
1. Read file line-by-line.
2. Track fence state via `FENCE_OPEN = r'^(\s*)```(!?)(\w*)\s*$'` and `FENCE_CLOSE = r'^(\s*)```\s*$'`.
3. A fence opened with `!` immediately after the opening triple-backticks is an INJECTION FENCE — every line inside is a detected injection.
4. A fence opened WITHOUT `!` (e.g., ` ```python `, ` ```yaml `, or bare ` ``` `) is an EXEMPTION ZONE — patterns inside are literal documentation.
5. Outside any fence, scan each line with `INLINE_INJECT = r'!`([^`\n]+)`'`. Every match is a detected injection site.
6. A detected injection site is ALLOWED if any of these are true:
   a. The file path + line number appears in the allowlist file (format: `<relpath>:<line>:<command-prefix>`). Match on path + line exact; command-prefix column is documentation/hint, not a strict check.
   b. The LINE IMMEDIATELY PRECEDING the injection contains an HTML comment matching `<!-- dc-shell-inject-ok: <reason> -->` where `<reason>` is non-empty (at least one non-whitespace character). Per D-02 inline marker.
7. On any non-allowed detected injection, print a diagnostic to stderr:
   ```
   validate-shell-inject: FAIL — unauthorized shell-inject in <file>:<lineno>:<col>
     pattern: !`<extracted command>`
     context: <line content, truncated to 120 chars>
     remediation: wrap in a non-`!` triple-backtick fence, add <!-- dc-shell-inject-ok: reason --> above it, or add <file>:<line>:<pattern> to scripts/shell-inject-allowlist.txt
   ```
   Exit 1 after emitting ALL violations (do not short-circuit on first — authors want a complete list).

8. On zero unauthorized detections, exit 0 silently (or print `validate-shell-inject: PASS <file>` to stderr when `--verbose` is passed).

- Bash outer wrapper (argparse, env gate, allowlist file resolution)
- `python3 - "$@" <<'PYEOF'` for the parser body (regex + state machine) — same pattern as `bin/dc-validate-synthesis.sh`
- Exit codes: 0 pass, 1 unauthorized injection found, 2 usage/file-not-found

#!/usr/bin/env bash
# scripts/test-shell-inject.sh — TD-04 fixture runner
# Exercises scripts/validate-shell-inject.sh against tests/fixtures/shell-inject/
# Each fixture has a pre-declared expected exit code (0 = parser passes, 1 = parser fails).
# Exit 0 on all fixtures matching expected; exit 1 on any mismatch.

set -euo pipefail

SCRIPT_DIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )" && pwd )"
REPO_ROOT="$( cd "$SCRIPT_DIR/.." && pwd )"
PARSER="$REPO_ROOT/scripts/validate-shell-inject.sh"
FIXTURE_DIR="$REPO_ROOT/tests/fixtures/shell-inject"

declare -a CASES=(
  "known-good-01.md:0"
  "regression-v1.0.0.md:1"
  "edge-inline-backtick.md:1"
  "edge-fenced-exempt.md:0"
  "edge-shebang.md:0"
  "allowlist-positive.md:0"

pass=0; fail=0
for case in "${CASES[@]}"; do
  fixture="${case%:*}"

  fi
done

printf "\n%s/%s fixtures passed\n" "$pass" "$((pass+fail))"
[ "$fail" -eq 0 ]
